## 5. Desktop client — contact documents UI

GUI components were added to `AddressPanel` (`.form`): a "Dokumente" tab, buttons
`cmdAdd` / `cmdAddFromTemplate`, table `tblContents`, popup `popDocuments`. Wiring is fully
programmatic — no further `.form` changes required.

- [x] 5.1 Add `AddressDocumentServiceRemote lookupAddressDocumentServiceRemote()` to
      `JLawyerServiceLocator`.
- [x] 5.2 `AddressDocumentStore extends ObservedDocumentStore` (client `launcher` package):
      saves changes of an opened document back via `setDocumentContent` (cheap local size check
      on close, no server round-trip), mirroring `CaseDocumentStore` without folders/tags/locking.
- [x] 5.3 `AddressPanel` wiring (`initDocumentsUI`, `loadAddressDocuments`, action handlers):
      - Table `tblContents` populated from `getAddressDocuments(addressId)` in `setAddressDTO`
        (columns Dokument / Datum / Größe; name column rendered with a format-dependent file-type
        icon; rows kept in `currentDocuments`).
      - **cmdAdd**: `JFileChooser` (multi-select) → `addDocument(...)`; **drag & drop** of OS
        files into the table via a `TransferHandler` (javaFileListFlavor).
      - **Double-click**: opens the document through `LauncherFactory` + `AddressDocumentStore`
        → edits are observed and written back to the server (like case documents).
      - **Delete**: via `popDocuments` context menu → `removeDocument(id)` (recycle bin).
      - New/unsaved contacts are guarded (`ensureAddressSaved`). Server enforces
        `readAddressRole`/`writeAddressRole`.
- [x] 5.4 **Create from template**: full dialog `AddAddressDocumentFromTemplateDialog`
      (based on `AddDocumentFromTemplateDialog`) — template tree/list, filename + name template,
      letterhead, editable placeholders table. Reuses the parties pipeline seeded with the single
      contact (case-less server call, `aFile=null`); the contact's reference type defaults to its
      `defaultRole` (matched by party-type name), else the first party type.
- [x] 5.5 **als PDF zum Kontakt speichern** (`mnuDuplicateDocumentAsPdf`): client-side PDF
      conversion via `FileConverter`, stored as a new address document (unique-name on conflict).
- [x] 5.6 **Umbenennen** (`mnuRenameDocument`): name prompt → `renameDocument(id, newName)`.
